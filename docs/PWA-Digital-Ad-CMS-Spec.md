# PWA 기반 디지털 광고 솔루션 + CMS 기술 스펙

## 1. 개요

- **PWA(Progressive Web App)**: 오프라인·저사양 환경에서도 동작하는 웹 기반 광고 플레이어/디스플레이
- **CMS(Content Management System)**: 광고 콘텐츠·캠페인·스케줄·미디어를 관리하는 백오피스
- **목표**: 대시보드/키오스크/디지털 사이니지용 단일 플랫폼

## 4. 기술 스펙

### 4.1 CMS (백오피스)

| 구분 | 기술 | 설명 |
|------|------|------|
| **프론트엔드** | React / Next.js 또는 Vue / Nuxt | SPA/SSR, 반응형 대시보드 |
| **상태 관리** | Redux / Zustand / Pinia | 캠페인/스케줄/미디어 상태 |
| **API 통신** | REST + (선택) GraphQL | 리소스 CRUD, 스케줄/캠페인 조회 |
| **백엔드** | **Python** (FastAPI / Django) 또는 Node.js (Nest/Express) | API 서버, 인증, 비즈니스 로직 |
| **DB** | PostgreSQL / MySQL | 사용자, 캠페인, 스케줄, 디바이스 메타데이터 |
| **파일 스토리지** | AWS S3 / Azure Blob / MinIO | 이미지, 동영상 업로드 |
| **CDN** | CloudFront / Cloudflare / Bunny CDN | 미디어 배포, 캐시 |
| **인증** | JWT + OAuth2 / Auth0 / Keycloak | 관리자 로그인, 역할(Admin/Editor/Viewer) |
| **실시간** | WebSocket / Server-Sent Events | 스케줄 푸시, 디바이스 상태 모니터링 |
| **스케줄링** | **Python**: Celery + Redis / **Node**: Bull / **기타**: Cron | 캠페인 활성화·비활성화, 리포트 집계 |
| **분석 저장** | PostgreSQL + 시계열 DB(선택: TimescaleDB/Influx) 또는 BigQuery | 노출/클릭/재생 이벤트 |

**백엔드 언어 선택 참고**

- **Python (FastAPI / Django)**: 빠른 개발, Celery로 배치·스케줄 작업에 강함, 데이터/리포트 처리에 유리. FastAPI는 비동기 API에 적합.
- **Node.js**: 프론트와 스택 통일, 실시간(WebSocket) 구현이 편함.

### 4.2 PWA 광고 플레이어

| 구분 | 기술 | 설명 |
|------|------|------|
| **프레임워크** | React / Vue / Vanilla JS | 단일 페이지, 풀스크린 재생 UI |
| **PWA** | Workbox (Service Worker) | 오프라인 캐시, 미디어/스케줄 캐시 전략 |
| **캐시 전략** | Cache First (미디어), Network First (스케줄 API) | 재생 안정성 vs 최신 스케줄 |
| **미디어** | HTML5 Video/Image, MSE(선택) | 비디오/이미지 슬라이드/HTML 위젯 |
| **동기화** | 주기적 폴링 (5~15분) 또는 WebSocket | 스케줄 갱신 |
| **식별** | 디바이스 ID (UUID 저장) + (선택) 그룹/위치 코드 | CMS에서 디바이스별 캠페인 배정 |
| **분석** | Beacon API / fetch 배치 | 노출/클릭/재생 완료 이벤트 전송 |
| **배포** | HTTPS 필수, CDN 정적 호스팅 | PWA 요구사항 |

### 4.3 인프라 및 공통

| 구분 | 기술 | 설명 |
|------|------|------|
| **호스팅** | AWS / GCP / Azure / Vercel + **백엔드 서버**(Python/Node/.NET 등) | CMS 앱 + API |
| **CI/CD** | GitHub Actions / GitLab CI / Jenkins | 빌드, 테스트, 배포 |
| **모니터링** | Sentry, DataDog, Prometheus + Grafana | 에러, API 지연, 디바이스 상태 |
| **보안** | HTTPS, CORS, API Rate Limit, 디바이스 토큰 | 무단 접근 방지 |

## 5. CMS 핵심 기능 목록

1. **사용자/권한**: 로그인, 역할(슈퍼관리자/캠페인관리자/뷰어), 팀/그룹
2. **미디어 라이브러리**: 업로드(이미지/동영상/HTML), 태그, 폴더, CDN URL 생성
3. **캠페인**: 이름, 기간, 타겟(디바이스 그룹/위치), 콘텐츠 목록, 우선순위
4. **스케줄**: 시간대별 재생 순서, 반복 요일, 긴급 배너 삽입
5. **디바이스/플레이어 관리**: 디바이스 등록, 그룹, 상태(온라인/오프라인/에러), 원격 설정
6. **템플릿/레이아웃**: 존 분할(풀/2분할/그리드), 위젯 타입(비디오/이미지/웹뷰)
7. **분석/리포트**: 노출수, 클릭, 재생률, 디바이스별/캠페인별 집계, 내보내기(CSV/Excel)
8. **API**: REST로 플레이어용 스케줄·콘텐츠 조회, 디바이스 인증, 이벤트 수신

## 6. PWA 플레이어 핵심 기능 목록

1. **설치**: 브라우저/홈화면 설치, 디바이스 ID 발급 및 CMS 등록
2. **스케줄 수신**: API 폴링 또는 WebSocket으로 재생 목록 수신
3. **오프라인 재생**: Service Worker로 미디어·스케줄 캐시, 네트워크 끊김 시 캐시 재생
4. **재생 엔진**: 비디오 루프, 이미지 슬라이드, 전환 효과, 타이머 기반 전환
5. **이벤트 수집**: 노출 시작/완료, 클릭(상호작용 시), 재생 에러, 오프라인 재생 여부
6. **원격 설정**: CMS에서 배정한 새 스케줄 반영, 강제 새로고침(선택)

---

## 7. 데이터 모델 (핵심 엔티티)

- **User**: id, email, role, team_id
- **Device**: id, device_id(UUID), name, group_id, location, last_seen, status
- **Campaign**: id, name, start_at, end_at, priority, target_group_ids
- **Content**: id, type(video/image/html), url, duration_sec, metadata
- **Schedule**: id, campaign_id, device_group_id, content_ids[], time_slots[], days_of_week
- **PlaybackEvent**: id, device_id, content_id, event_type(impression/click/complete/error), at

---

## 8. API 예시 (플레이어용)

- `POST /api/device/register` — 디바이스 등록, device_id 발급
- `GET /api/player/schedule?device_id=xxx` — 해당 디바이스 스케줄 + 콘텐츠 URL
- `POST /api/events` — 이벤트 배치 전송 (노출/클릭/완료/에러)

## 9. 레이아웃 및 화면 구성

### 9.1 CMS 백오피스 화면 구성

**전체 레이아웃**

- **고정 영역**: 좌측 사이드바(메뉴) + 상단 헤더(로그아웃, 알림, 프로필)
- **가변 영역**: 메인 컨텐츠(테이블/폼/차트). 반응형으로 태블릿·모바일에서 사이드바 접기 또는 햄버거 메뉴.

```
┌─────────────────────────────────────────────────────────────────────────┐
│  [로고]  대시보드  캠페인  미디어  스케줄  디바이스  리포트  설정    [🔔][👤]  │
├──────────┬──────────────────────────────────────────────────────────────┤
│          │                                                              │
│  대시보드  │                    메인 컨텐츠 영역                          │
│  캠페인   │                    (테이블 / 폼 / 차트 / 미리보기)              │
│  미디어   │                                                              │
│  스케줄   │                                                              │
│  디바이스  │                                                              │
│  리포트   │                                                              │
│  설정     │                                                              │
│          │                                                              │
└──────────┴──────────────────────────────────────────────────────────────┘
```

**주요 화면**

| 화면 | 구성 요소 | 비고 |
|------|-----------|------|
| **로그인** | 이메일/비밀번호, (선택) 소셜 로그인 | 세션·JWT 저장 |
| **대시보드** | KPI 카드(캠페인 수, 디바이스 수, 오늘 노출 등), 최근 캠페인 목록, 디바이스 상태 요약, 간단 차트 | 한눈에 현황 파악 |
| **캠페인 목록** | 테이블(이름, 기간, 상태, 조치), 필터/검색, [새 캠페인] 버튼 | |
| **캠페인 상세/편집** | 기본 정보 폼, 콘텐츠 드래그 앤 드롭 목록, 타겟(디바이스 그룹) 선택, 미리보기(선택) | 단계별 위저드도 가능 |
| **미디어 라이브러리** | 그리드/리스트 뷰, 폴더 트리, 업로드 영역, 미디어 카드(썸네일, 이름, 길이, 삭제/편집) | 대용량 업로드·진행률 표시 |
| **스케줄** | 캠페인별·디바이스 그룹별 탭, 타임라인/캘린더 뷰, 시간대별 슬롯에 콘텐츠 배치 | 드래그로 순서 변경 |
| **디바이스 목록** | 테이블(이름, 그룹, 상태, 마지막 접속), 그룹 필터, [등록] 버튼 | 상태: 온라인/오프라인/에러 |
| **디바이스 상세** | 정보 편집, 배정된 캠페인/스케줄, 최근 이벤트 로그, 원격 새로고침(선택) | |
| **리포트** | 기간·캠페인·디바이스 필터, 노출/클릭/재생률 차트, 테이블, [CSV/Excel 내보내기] | |

### 9.2 디스플레이(플레이어) 레이아웃

**화면을 여러 “존(Zone)”으로 나누어, 존별로 다른 콘텐츠를 재생할 수 있게 구성한다.**

**레이아웃 템플릿 종류**

| 템플릿 ID | 설명 | 비율 예시 | 용도 |
|-----------|------|-----------|------|
| **full** | 단일 존, 풀스크린 | 100% | 단일 영상/이미지 슬라이드 |
| **split_h** | 좌·우 2분할 | 50 : 50 (또는 60 : 40 등) | 메인 영상 + 사이드 배너/텍스트 |
| **split_v** | 상·하 2분할 | 50 : 50 | 상단 배너 + 하단 메인 |
| **triple_h** | 가로 3등분 | 33 : 33 : 33 | 메인 + 보조 x2 |
| **grid_2x2** | 2x2 그리드 | 4개 동일 | 4개 콘텐츠 동시 재생 |
| **main_side** | 메인(큰) + 사이드(작은) | 70 : 30 | 메인 영상 + 날씨/뉴스 등 |

**존별 콘텐츠 타입**

- **video**: 비디오 루프 재생
- **image**: 이미지 슬라이드(전환 효과, 재생 시간 설정)
- **html**: iframe 또는 HTML 스니펫(날씨, 시계, RSS 등)
- **placeholder**: 대기 화면 또는 로고

**레이아웃 정의 예시 (JSON)**

```json
{
  "layout_id": "split_h",
  "zones": [
    { "id": "zone_1", "ratio": 0.6, "content_type": "video" },
    { "id": "zone_2", "ratio": 0.4, "content_type": "image" }
  ]
}
```

**플레이어 화면 예시 (2분할)**

```
┌─────────────────────────────────────┬─────────────────────┐
│                                     │                     │
│           Zone 1 (메인)              │    Zone 2 (사이드)   │
│           비디오 / 이미지            │    이미지 슬라이드   │
│                                     │    또는 HTML 위젯   │
│                                     │                     │
└─────────────────────────────────────┴─────────────────────┘
```

- CMS에서 **캠페인 또는 스케줄**에 `layout_id`와 **존별 콘텐츠 목록**을 지정.
- 플레이어는 스케줄 API에서 레이아웃 + 존별 미디어 URL을 받아, CSS Grid/Flex로 영역을 나누고 각 존을 독립 재생.
